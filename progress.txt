## Issue #21 — schema-drift guard between packages/schema and ops's vendored copy
- Urgency raised because PR #40 (open, adds `brand.motion`) is the first real
  canonical schema change since the freeze; no mechanism existed to catch a
  missed ops re-sync.
- ops repo isn't accessible from here, so per the issue's own fallback plan
  I built a vendored shape-snapshot mechanism entirely within this repo:
  - `packages/schema/src/shape.ts`: normalizes a Zod schema into a
    comparable, JSON-serializable shape via `_def` introspection (field
    keys/nesting/kind — object/array/record/enum/literal/primitive),
    deliberately dropping check-level detail (regex, min/max) that
    legitimately differs from ops's hand-converted `.mjs` copy. Also
    `presetsShape()` for `presets.ts`'s plain id lists/token maps, and a
    generic `diffShapes()` deep-diff.
  - `packages/schema/ops-shape.snapshot.json`: frozen snapshot of the shape,
    representing the last known-synced `ops/lib/schema`. Generated by
    `pnpm schema:snapshot-ops` (packages/schema/scripts/refresh-ops-snapshot.ts,
    root package.json script since tsx only lives at root, matching
    new-client/eject-client's pattern).
  - `packages/schema/src/ops-drift.test.ts`: 3 vitest tests — ClientConfig
    shape matches snapshot, presets shape matches snapshot, and a self-test
    that seeds a `brand.motion`-style field-only-here-not-in-snapshot drift
    and asserts diffShapes catches it (the DoD's fixture requirement).
  - Wired into the gate for free: `ralph/gate.sh` already runs
    `turbo run test`, which now picks up the schema package's new test file
    — no ci.yml changes needed.
  - Documented in AGENTS.md § "Contracts this repo owns": what the guard
    does, how to refresh the snapshot, and the rule to sync ops first rather
    than regenerate the snapshot to silence a real drift.
- Verified end-to-end: manually deleted a field from the live
  ops-shape.snapshot.json, confirmed the test failed with a readable diff +
  re-sync hint, then restored it and confirmed green again.
- Gate green: 15/15 turbo tasks (build + check + test).
